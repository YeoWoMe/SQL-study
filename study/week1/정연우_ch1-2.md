### 데이터베이스(Database, DB)

- 논리적으로 연관된 데이터를 모아 일정한 형태로 저장해 놓은 것
- 중복 최소화 & 조직의 목적에 맞게 관리

<aside>
📌

**ISOS**

---

- **Integrated** data: 데이터 중복을 최소화
- **Stored** data: 컴퓨터가 접근할 수 있는 저장 매체에 저장된 데이터
- **Operational** data: 조직의 업무를 수행하는 데 반드시 필요한 데이터
- **Shared** data: 여러 응용 시스템이 공동으로 소유, 유지하는 데이터
</aside>

<aside>
📌

**DB의 특징: R1C3**

---

- Real time accessibility: 사용자의 질의에 **실시간** 응답으로 처리
- Continuous evolution: 삽입, 삭제, 수정(insertion, deletion, modification) 작업을 하여 항상 **최신 데이터를 동적으로 유지**
- Concurrent sharing: 목적이 서로 다른 여러 사용자가 동시에 원하는 데이터를 공유
- Content reference: DB 내 데이터를 참고할 때, 사용자가 요구하는 **데이터 내용**을 참조
</aside>

**왜 DB가 필요할까?**

**⇒ 데이터 중복을 최소화하고 조직의 목적에 맞게 데이터를 효율적으로 관리할 수 있는 수단**!

### 데이터베이스 관리 시스템(Database Management System, DBMS)

- 데이터베이스 관리 프로그램으로, 데이터를 입력, 수정, 삭제 등의 기능을 제공
- 데이터베이스 내에 있는 데이터를 사용자의 요청에 따라 처리할 수 있도록 하는 프로그램

**계층형 데이터베이스(hierarchical database)**

- 데이터가 부모와 자식 관계인 트리 구조
- 데이터를 수정하기 용이하지 않아 사용되지 않는 경우가 많음

**네트워크형 데이터베이스(network database)**

데이터를 노드로 표현한 모델로, 각 노드는 서로 대등한 관계로 연결되어 있음

**키-값 데이터베이스(key-value database)**

NoSQL의 한 종류로, key-value를 1-1 대응해 데이터를 저장

JSON, XML schema 등으로 키와 값이 쌍으로 이루어지면 키-값 데이터베이스라고 칭함

**관계형 데이터베이스(relational database)**

데이터를 행과 열로 구성한 **테이블**로 정리하고, 테이블 간에 관계를 가지고 있음

Primary key(PK)가 각 행을 식별하고, 열 속성(feature)에 따라 데이터의 유형(날짜, text, Boolean 등)이 정해짐

### SQL(Structure Query Language)

- 데이터베이스에서 데이터의 조작과 정의를 하기 위해 사용하는 **언어**
- 데이터베이스에 데이터를 입력/조회/수정/삭제(Create, Read, Update, Delete, 이하 CRUD)를 하기 위해 사용
- 각 vendor 사마다 ANSI SQL(국제 통용 SQL)를 따르면서 제품에 특화된 문법이나 명세를 추가

**SQL 문법**

- 데이터 정의 언어(Data Define Language, DDL): 테이블 및 스키마를 관리
- 데이터 조작 언어(Data Manipulation Language, DML): 데이터를 검색, 입력, 수정, 삭제하는 언어
- 데이터 제어 언어(Data Control Language, DCL): 데이터베이스 접근 및 권한 부여
